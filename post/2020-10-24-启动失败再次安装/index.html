启动失败再次安装

 2020-10-24 745 字 安装 

启动服务、增加post用Addins菜单。（好久不用容易忘）

前几天对所有项目目录进行了整理，调整了blogs的目录位置。然后……

启动失败，只能重新来过，以下是重装过程。

周六早晨

  • 打开 blogdown书和我的第一篇安装文章。
  • 检查已安装的Hugo版本，blogdown::hugo_version() [1] ‘0.74.3’，升级Hugo，blogdown::update_hugo() ，发现安装位置不是我想要的。
  • 直接从 github 的 gohugoio/hugo 下载 hugo_extended_0.76.5_Windows-64bit.zip，修改安装位置的文件路径名，hugo_0743→hugo，修改系统path。
  • 新建一个目录，然后按第一篇文章步骤安装。
  • 新建project，新建站点时采用blogdown::new_site(theme = 'amzrk2/hugo-theme-fuji')直接用fuji界面。
  • 老的content目录中文件导入新目录，新的content目录中不要的文件删除。
  • 修改根目录中index.Rmd和config.toml两个文件。
  • OK。

周六下午

  • 吃完午饭又启动不了了（在Rstudio中）。
  • bing：It took more than 30 seconds to launch the server. There may be something wrong. The process has been killed. If the site needs more time to be built and launched, set options(blogdown.server.timeout) to a larger value. 发现有人有相同问题
  • 在cmd中，到blog本地路径运行：hugo server 可以启动，且Rstudio中更新可在浏览器中实时反馈。
  • 查询搜索，似乎与R（V4.03）和Rstudio（V1.4.953）版本更新有关。
  • public目录下没有文件！？文件到哪里去了？
  • 改回R（V4.02）未解决问题。
  • 改回Rstudio（V1.3.1093）未解决问题。
  • 待续……，先放着吧。

另外

要好好学习一下github ，把blog上传到github上去，不要每次都用COPY。
